## End-to-End Flows

All 8 E2E flows complete successfully.

1. **Dynamic INT8 quantization via Python API** ✅
   - `quantize(model, bits=8, dynamic=True)` → `QuantizationResult`
   - Result includes `.model`, `.save()`, `.validate()` methods

2. **Static INT4 quantization via Python API** ✅
   - `quantize(model, bits=4, calibration_data=data)`
   - INT4 with group-wise scaling, layer skipping, accuracy warnings

3. **Calibration with advanced observer** ✅
   - `create_observer('Histogram', bins=1024)` → calibration → quantization
   - Factory supports 'minmax', 'movingaverage', 'histogram', 'auto'

4. **Model save/load round-trip** ✅
   - Quantize → save (.pt/.safetensors) → load → validate
   - Full round-trip tested: 37.32x compression achieved

5. **Layer skipping workflow (INT4)** ✅
   - `DEFAULT_INT4_SKIP` protects embeddings, normalization, small layers
   - 512 parameter threshold automatically applied

6. **Accuracy warnings system** ✅
   - SQNR thresholds: <10 dB (critical), <20 dB (warning), >30 dB (good)
   - Integrated into `static_quantize` output

7. **CLI → API integration** ✅
   - `monoquant quantize --model model.pt --bits 8 --dynamic`
   - CLI imports `quantize` from API, handles errors

8. **QuantizationResult methods** ✅
   - `.save()` calls `save_model()`
   - `.validate()` calls `validate_quantization()`

## Key Achievements

1. **Model-Agnostic Design**: Works with any PyTorch model - HuggingFace, local, custom
2. **Minimal Dependencies**: Only torch required (safetensors, click, tqdm are optional/dev deps)
3. **Dual Interface**: Python API for automation, CLI for CI/CD
4. **Robust Calibration**: MinMax, MovingAverage, Histogram observers
5. **INT4 Support**: Group-wise scaling with packed storage (2x compression vs INT8)
6. **Layer Protection**: Default skip list for sensitive layers
7. **Validation**: SQNR metrics, size comparison, load testing
8. **Serialization**: PyTorch and Safetensors format support
